Add-in: Legislative Report Router

Workflow

  1. Open (or compose) the Daily Bill Report and click Report Router on the ribbon.
  2. One-time Settings: cloud environment, SharePoint site URL, then Connect & load routing rules.
  3. Parse Report → review each bill's divisions and routes → Preview → Publish (explicit confirmation required).

Common issues

  • Unknown division — no active rule matches. Fix the division text, add an alias to the routing list, or exclude the bill.
  • Route has no tag — the post publishes without a mention; add the TeamsTagId (Settings → tag lookup) to mention the tag.
  • Duplicate protection — re-publishing a report skips bills already recorded as published in the audit list; only failures re-run.
  • Bills not detected — the bill number must appear alone on its own line ("HF935"). See the parser rules in the repository docs.

Privacy

No data collection — parsing and publishing run in your own Microsoft 365 session. See the Privacy Policy.
